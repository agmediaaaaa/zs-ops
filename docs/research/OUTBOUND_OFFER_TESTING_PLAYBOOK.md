# Outbound Offer Testing Playbook

Scaling / testing method for systematically identifying which offer angles perform best across ICP segments and scaling the winners.

---

## Objective

Systematically identify which **offer angles perform best across different ICP segments** by running structured outbound campaigns and scaling the winning offers.

This framework removes guesswork by testing multiple offers simultaneously and letting **market response determine the winning positioning**.

---

# 1. Separate Campaign Systems

Outbound activity runs through **two independent campaign tracks**.

### Track A — Signal / Intent Campaigns

Handled separately from the testing engine.

Examples of triggers:

- Open jobs
- Leadership changes
- Funding events
- Rapid company growth

These campaigns run continuously and are **not part of the offer-testing framework**.

---

### Track B — ICP Offer Testing Engine

This system is designed to discover **which offer angles resonate with specific ICP segments**.

The structure:

```
ICP → Offer → Campaign → Lead List → Test
```

Each campaign isolates **one offer angle** for a specific ICP.

---

# 2. ICP Segmentation

Define **three major ICP groups** based on company maturity or structural differences.

Example segmentation:

```
ICP 1
ICP 2
ICP 3
```

These ICPs can represent **any segmentation criteria**, such as:

- Company size
- Stage
- Industry group
- Organizational structure
- Geography
- Operational maturity

The specific definition of each ICP is flexible and should reflect **meaningful differences in buying behavior**.

---

# 3. Offer Distribution Per ICP

Each ICP runs **five separate offer campaigns**.

Structure:

```
ICP
 ├ Offer 1
 ├ Offer 2
 ├ Offer 3
 ├ Offer 4
 └ Offer 5
```

Each offer represents a **different positioning or value proposition**.

Each offer is tested through **its own outbound campaign**.

---

# 4. Lead Allocation Per Campaign

Each campaign receives:

```
1000 – 1500 leads
```

This volume ensures enough outreach to produce **reliable response signals**.

---

# 5. Total Outreach Per ICP

Per ICP:

```
5 offers × 1000–1500 leads = 5000–7500 contacts
```

---

# 6. Total System Volume

Across all ICP segments:

```
3 ICPs × 5000–7500 contacts = 15,000 – 22,500 contacts
```

This total outreach volume provides sufficient scale to evaluate **multiple offers simultaneously**.

---

# 7. Campaign Structure

Each campaign tests **one variable only**: **the offer angle**.

Everything else should remain consistent:

- ICP criteria
- Industry filters
- Geography
- Messaging structure
- Outreach format

This isolates the impact of the **offer itself**.

---

# 8. Lead List Rules

Each campaign must use a **distinct lead list**.

Requirements:

- Lists must **not overlap** between campaigns
- Leads must match the same ICP definition
- Lists must be large enough to supply 1000–1500 contacts

Example structure:

```
ICP Lead Pool
    ↓
Split into 5 separate campaign lists
```

---

# 9. Campaign Launch

All campaigns are launched **at roughly the same time**.

Advantages:

- Rapid feedback
- Comparable performance across offers
- Faster identification of winners

---

# 10. Evaluation Window

Campaign results should be evaluated after:

```
5–7 days
```

This timeframe allows enough outreach to produce **stable performance signals**.

---

# 11. Metrics to Track

Evaluate each campaign using three primary metrics.

### Reply Rate

Measures overall engagement.

Typical outbound range: **1.2 – 3%**

---

### Positive Reply Rate

Measures genuine interest.

Typical range: **10–30%** (of replies)

---

### Meeting Conversion

Measures ability to convert interest into meetings.

Typical range: **20–40% of positive replies**

---

# 12. Determining the Winning Offer

Compare the results of the five campaigns within the same ICP.

Example comparison:

| Offer   | Reply Rate | Positive Replies |
| ------- | ---------- | ---------------- |
| Offer 1 | X          | X                |
| Offer 2 | X          | X                |
| Offer 3 | X          | X                |
| Offer 4 | X          | X                |
| Offer 5 | X          | X                |

The offer producing the **highest positive reply rate and meeting conversions** becomes the winning offer.

---

# 13. Scaling the Winner

Once the best-performing offer is identified:

1. Expand the lead pool
2. Increase sending volume
3. Refine messaging variations

Example scaling path:

```
Winning Offer
    ↓
Expand to full ICP list
    ↓
Increase outbound volume
    ↓
Optimize messaging
```

---

# 14. Iteration Cycle

After scaling the winning offer:

- Continue testing additional offers
- Refine messaging
- Expand ICP coverage

This turns outbound into a **continuous experimentation system**.

---

# 15. Final Architecture Overview

```
Intent Campaigns (separate)
• Open jobs
• Leadership changes
• Funding
• Hypergrowth

Offer Testing Engine
3 ICP segments
× 5 offers per ICP
× 1000–1500 leads per offer
```

**Total campaigns:** 15 campaigns | 15k–22k contacts

Winning offers are then **scaled across the full market**.
